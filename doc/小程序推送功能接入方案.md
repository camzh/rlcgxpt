# 小程序推送功能接入方案

## 一、推送类型总览

| 推送场景 | 触发时机 | 推送对象 |
|----------|----------|----------|
| 审批通知 | 有待审批事项时 | 对应管理员/超级管理员 |
| 货源/需求更新通知 | 任何货源/需求新增/状态变更时 | 所有已认证用户（含管理员） |
| 跟进提醒 | 跟进满1小时未更新状态时 | 跟进人 + 其组管理员 |
| 周报推送 | 每周一早上9:00 | 所有用户（按角色生成不同报告） |

## 二、微信订阅消息机制

### 2.1 核心原理

微信订阅消息（SubscribeMessage）的限制：
- **必须用户主动授权**：每次推送前需调用 `wx.requestSubscribeMessage` 获取用户许可
- **有效期有限**：一次性订阅仅当次有效，长期订阅需用户勾选"总是保持以上选择"
- **模板消息 vs 订阅消息**：本方案使用订阅消息，模板消息（form_id）已不推荐

### 2.2 用户授权时机

建议在以下用户操作时静默请求授权（不在操作流程中打断用户）：

1. **用户首次打开小程序** → 弹窗"开启消息通知，审批和跟进不再错过"
2. **用户点击"跟进"按钮时** → 同时请求"跟进提醒"相关模板授权
3. **用户提交审核时** → 同时请求"审批结果通知"授权

```javascript
// 统一封装授权函数
function requestMessageSubscription(tmplIds) {
  return new Promise((resolve) => {
    wx.getSetting({
      withSubscriptions: true,
      success: (res) => {
        const maintained = res.subscriptionsSetting?.mainSettings || {};
        const missing = tmplIds.filter(id => !maintained[id]);
        if (missing.length === 0) {
          resolve(true); // 已全部授权
          return;
        }
        wx.requestSubscribeMessage({
          tmplIds: missing,
          success: () => resolve(true),
          fail: () => resolve(false)
        });
      },
      fail: () => resolve(false)
    });
  });
}
```

## 三、消息模板配置（需在小程序后台操作）

登录 **微信公众平台 → 设置与服务 → 订阅消息 → 添加模板**，按以下关键词组合申请：

### 3.1 模板一：审批通知
- 模板标题：`审核结果通知`
- 关键词：审批类型、申请人、审批状态、审批时间

### 3.2 模板二：更新通知
- 模板标题：`库存动态提醒`
- 关键词：标题、内容摘要、更新时间、操作人

### 3.3 模板三：跟进提醒
- 模板标题：`跟进提醒`
- 关键词：跟进人、货源/需求标题、提醒时间、待处理事项

### 3.4 模板四：周报推送
- 模板标题：`业绩周报`
- 关键词：姓名、周期、数据摘要、查看详情链接

申请通过后，记录每个模板的 **TemplateId**，填入后端配置文件。

## 四、后端 API 接口设计

### 4.1 发送订阅消息

**接口**：POST `/api/mini/subscribe/send`

**请求参数**：
```json
{
  "openid": "oXXXXX",
  "templateId": "TM0001",
  "page": "pages/detail/index?id=xxx&type=supply",
  "data": {
    "thing1": { "value": "审批通知" },
    "thing2": { "value": "李诚提交了货源上传申请" },
    "phrase3": { "value": "待审核" },
    "date4": { "value": "2026-05-30 21:30" }
  }
}
```

**实现要点**：
- 调用微信 `POST https://api.weixin.qq.com/cgi-bin/message/subscribe/bizsend`
- 需要 `access_token`（7200秒有效期，需缓存并自动刷新）
- `openid` 需通过 `wx.login` + `code2session` 获取，并存储在用户记录中

### 4.2 消息触发场景对应关系

| 业务事件 | 触发函数 | 通知对象 | 模板ID |
|----------|----------|----------|--------|
| 注册审批申请 | `approveRegistrationRequest` | 对应组管理员 | TM0001 |
| 货源上传待审核 | `requestCompleteItem` | 组管理员 | TM0001 |
| 需求上传待审核 | `requestCompleteDemand` | 组管理员 | TM0001 |
| 完成审核通过 | `approveCompletionRequestToCloud` | 申请人 + 组管理员 | TM0001 |
| 下架审核通过 | `approveOfflineRequestToCloud` | 申请人 | TM0001 |
| 新增/更新货源/需求 | `upsertItemToCloud` | 所有用户 | TM0002 |
| 跟进满1小时 | 定时任务 | 跟进人 + 组管理员 | TM0003 |
| 周报推送 | 定时任务（每周一9:00） | 所有用户 | TM0004 |

## 五、跟进提醒实现方案（1小时自动推送）

### 5.1 字段准备

货源/需求数据需新增字段：
```javascript
// 货源
followOwnerId: string,       // 跟进人ID
followOwnerName: string,      // 跟进人姓名
lastFollowedAt: string,       // ISO时间，标记跟进的时间点
followReminderSent: boolean,  // 是否已发送提醒，避免重复推送
followOwnerGroup: string      // 跟进人所在组，用于查找组管理员
```

### 5.2 标记跟进时写入时间戳

```javascript
// 在 inventory.js 的 updateStatus 函数中
// 当 status → FOLLOWING 时
if (nextStatus === INVENTORY_STATUS.FOLLOWING) {
  next.followOwnerId = user.id;
  next.followOwnerName = user.name;
  next.lastFollowedAt = new Date().toISOString();  // ← 新增
  next.followReminderSent = false;                   // ← 新增
  next.followOwnerGroup = user.group || "";        // ← 新增
}
```

### 5.3 定时检查逻辑（定时任务）

```javascript
// 云函数：checkFollowReminders（每30分钟触发一次）
async function checkFollowReminders() {
  const now = Date.now();
  const ONE_HOUR = 60 * 60 * 1000;

  // 找出超时的跟进记录
  const overdueItems = getItems().filter(item => {
    if (!item.lastFollowedAt || item.followReminderSent) return false;
    const elapsed = now - new Date(item.lastFollowedAt).getTime();
    return elapsed > ONE_HOUR && item.status === INVENTORY_STATUS.FOLLOWING;
  });

  const overdueDemands = getDemands().filter(item => {
    if (!item.lastFollowedAt || item.followReminderSent) return false;
    const elapsed = now - new Date(item.lastFollowedAt).getTime();
    return elapsed > ONE_HOUR && item.status === "following";
  });

  for (const item of [...overdueItems, ...overdueDemands]) {
    const user = getUserById(item.followOwnerId);
    if (!user) continue;

    // 发送消息给跟进人
    if (user.openid) {
      await sendSubscribeMessage(user.openid, TEMPLATE_IDS.FOLLOW_REMINDER, {
        thing1: { value: user.name },
        thing2: { value: item.title },
        date3: { value: formatDateTime(new Date()) },
        thing4: { value: "已跟进超过1小时，请及时更新状态" }
      });
    }

    // 发送消息给组管理员
    const admin = getGroupAdmin(user.group);
    if (admin && admin.openid) {
      await sendSubscribeMessage(admin.openid, TEMPLATE_IDS.FOLLOW_REMINDER, {
        thing1: { value: user.name },
        thing2: { value: item.title },
        date3: { value: formatDateTime(new Date()) },
        thing4: { value: "已跟进超过1小时，请督促更新状态" }
      });
    }

    // 标记已发送
    if (item.side === "supply") {
      updateItemReminderSent(item.id);
    } else {
      updateDemandReminderSent(item.id);
    }
  }
}
```

## 六、周报自动生成方案

### 6.1 数据统计口径

**个人周报**（每周一推送）：
- 本周新增发布数（supply + demand）
- 本周成交数（status=sold/done）
- 正在跟进数（status=following）
- 成交率 = 成交数 / 总发布数
- 相比上周变化（增减百分点）

**管理员/超级管理员周报**：
- 包含上述个人数据
- 团队汇总：本周团队总发布、成交、跟进
- 待审批数：注册待审批 + 货源/需求待审批
- 新增注册人数
- 逾期未更新记录（跟进超过7天无操作）

### 6.2 周报生成函数

```javascript
async function generateWeeklyReports() {
  const users = getApprovedUsers();
  const weekStart = getWeekStartDate(); // 本周一 00:00
  const weekEnd = new Date();           // 当前时间

  for (const user of users) {
    const report = buildWeeklyReport(user, weekStart, weekEnd);

    if (user.role === "staff") {
      const personal = buildPersonalReport(user, weekStart, weekEnd, report);
      if (user.openid) {
        await sendSubscribeMessage(user.openid, TEMPLATE_IDS.WEEKLY_REPORT, {
          thing1: { value: user.name },
          date2: { value: `${formatDate(weekStart)}~${formatDate(weekEnd)}` },
          thing3: { value: `发布${personal.publish} | 成交${personal.sold} | 跟进${personal.following}` },
          thing4: { value: personal.rateChange >= 0 ? `成交率↑${personal.rateChange}%` : `成交率↓${Math.abs(personal.rateChange)}%` }
        });
      }
    } else if (user.role === "admin" || user.role === "superadmin") {
      const business = buildBusinessReport(user, weekStart, weekEnd, report);
      if (user.openid) {
        await sendSubscribeMessage(user.openid, TEMPLATE_IDS.WEEKLY_REPORT, {
          thing1: { value: user.name },
          date2: { value: `${formatDate(weekStart)}~${formatDate(weekEnd)}` },
          thing3: { value: `团队发布${business.teamPublish} | 团队成交${business.teamSold} | 待审${business.pendingCount}` },
          thing4: { value: `新增注册${business.newUsers}人` }
        });
      }
    }
  }
}
```

## 七、access_token 管理

微信 access_token 是调用所有微信 API 的凭证，有效期7200秒，必须：

1. **集中管理**：在服务端（或云函数）统一获取和刷新
2. **缓存使用**：缓存至 Redis 或全局变量，快过期时自动刷新
3. **绝不泄露 appsecret**：appsecret 只存在于后端，绝不传送到小程序前端

```javascript
// 云函数：getAccessToken
let cachedToken = null;
let tokenExpireAt = 0;

async function getAccessToken() {
  if (cachedToken && Date.now() < tokenExpireAt - 60000) {
    return cachedToken; // 提前1分钟刷新
  }
  const res = await wx.cloud.callContainer({
    config: { env: process.env.CLOUD_ENV },
    path: '/api/token',
    method: 'GET'
  });
  cachedToken = res.access_token;
  tokenExpireAt = Date.now() + (res.expires_in || 7200) * 1000;
  return cachedToken;
}
```

## 八、openid 获取与存储

用户首次使用时通过 `wx.login` 获取 code，换取 openid 并存储：

```javascript
// 小程序端
wx.login({
  success: async (res) => {
    if (!res.code) return;
    // 发送给后端换取 openid
    const result = await wx.request({
      url: 'https://21068849.xyz/api/mini/code2session',
      data: { code: res.code }
    });
    if (result.openid) {
      // 更新用户记录，存储 openid
      updateUserOpenid(app.globalData.activeUserId, result.openid);
    }
  }
});
```
